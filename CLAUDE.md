# Claude Code Configuration: cobol-legacy-ledger

## Project Overview

Non-invasive cryptographic integrity layer for inter-bank COBOL batch settlement. Wraps unmodified COBOL with Python observation and SHA-256 hash chain verification. 6 nodes (5 banks + clearing house), 3-phase delivery.

**Interview narrative**: "COBOL isn't the problem. Lack of observability is."

## Critical Principles (See memory/constitution.md)

1. **COBOL Immutability** — Wrap, don't modify. Legacy systems unchanged.
2. **Cryptographic Integrity** — SHA-256 chain + HMAC. Detect tampering in <100ms.
3. **Per-Node Database** — Each node has bank_a.db, clearing.db, etc. No shared ledger.
4. **Specification-Driven** — All COBOL implementation follows 02_COBOL_AND_DATA.md exactly.
5. **6-Node Architecture** — Fixed: BANK_A, BANK_B, BANK_C, BANK_D, BANK_E, CLEARING.
6. **Production COBOL** — Headers + KNOWN_ISSUES.md documentation required.
7. **Phase Gates** — Phase 1 gate (5 checks) blocks Phase 2.
8. **Testability** — Every requirement must be testable.
9. **No Node.js** — Static HTML/CSS/JS only. No npm, no build process.
10. **Clear Error Paths** — Status codes: 00=success, 01=NSF, 02=limit, 03=invalid, 04=frozen, 99=error.

```bash
# 1. COBOL compiles (or gracefully skips)
./scripts/build.sh

# 2. All 6 nodes seed
./scripts/seed.sh

# 3. Bridge reads correct account counts
python -c "from bridge import COBOLBridge; ..."  # BANK_A:8, BANK_B:7, BANK_C:8, BANK_D:6, BANK_E:8, CLEARING:5

# 4. Transaction processes with status 00
python -c "from bridge import COBOLBridge; result = b.process_transaction(...); assert result['status'] == '00'"

# 5. Integrity chain records and verifies
python -c "from bridge import COBOLBridge; verify = b.chain.verify_chain(); assert verify['valid']"
```

### Record Formats

**ACCTREC (Account)**: 70 bytes
- ACCT-ID (10) | ACCT-NAME (30) | ACCT-TYPE (1) | ACCT-BALANCE (12, implied decimal) | ACCT-STATUS (1) | ACCT-OPEN-DATE (8) | ACCT-LAST-ACTIVITY (8)

**TRANSREC (Transaction)**: 103 bytes
- TRANS-ID (12) | TRANS-ACCT-ID (10) | TRANS-TYPE (1) | TRANS-AMOUNT (12, implied decimal) | TRANS-DATE (8) | TRANS-TIME (6) | TRANS-DESC (40) | TRANS-STATUS (2) | TRANS-BATCH-ID (12)

**BATCH-INPUT**: Pipe-delimited
- Format: `ACCOUNT_ID|TYPE|AMOUNT|DESCRIPTION` (D/W/I/F)
- Format: `ACCOUNT_ID|T|AMOUNT|DESCRIPTION|TARGET_ID` (transfers only)

## Troubleshooting

**Q: Where do I start?**
A: Read the handoff documents in order (00 → 01 → 02 → 03 → 04). Then run `/speckit.clarify`.

**Q: Can I copy COBOL from somewhere?**
A: No. Implement from the specification in 02_COBOL_AND_DATA.md. The spec is complete and testable.

**Q: What if cobc isn't installed?**
A: Phase 1 gate handles it gracefully. build.sh skips compilation, seed.sh uses Python file I/O (Mode B), system still works.

**Q: Why per-node databases?**
A: Financial systems are distributed. Each node operates independently. Per-node isolation enables Phase 2 cross-node verification where the clearing house proves what should have happened.

**Q: What's the interview pitch?**
A: "COBOL isn't the problem. Lack of observability is. You can build modern infrastructure around legacy systems without replacing them." See principle #11 in constitution.md.
